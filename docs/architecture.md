# Архитектура

## Обзор

Проект построен вокруг Wails‑приложения:

- Backend на Go предоставляет API для UI
- Frontend на Vite обращается к методам через `wailsjs`
- Аудио‑файлы отдаются через встроенный HTTP‑обработчик `/audio/...`

## Ключевые модули

- `main.go` — инициализация Wails и подключение `AudioHandler`
- `app.go` — методы, доступные фронтенду
- `config/` — хранение и валидация пути к папке музыки
- `library/` — сканирование плейлистов и аудио
- `media/` — чтение метаданных и обложек
- `server/` — безопасная раздача файлов по `/audio/...`
- `frontend/` — UI и сборка через Vite

## Потоки данных

1. UI вызывает `SetMusicFolder`.
2. Backend сохраняет путь в конфиг.
3. UI вызывает `GetPlaylists` и `ScanAudioFiles`.
4. Для воспроизведения UI строит URL `/audio/<relative-path>`.
5. `AudioHandler` проверяет путь и отдаёт файл.
